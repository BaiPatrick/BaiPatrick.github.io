前言

个人网站在面试公司的时候会加分，而且能够锻炼自己的写作、语言组织等能力，好处多多。我来教大家如何使用hexo搭建个人博客以及搭建过程中一些问题的解决方法。文章末尾有各种博客主题，如有描述不当的请各位多多指正。

准备工具

一、nodejs搭建

  1. 下载

    下载地址：nodejs下载

  2. 安装

    安装程序

    下载完成后，双击安装包，开始安装，使用默认配置一直点next即可，自定义修改安装路径。


    安装路径默认在C:\Program Files下面，也能够自定义修改，而后点击next

    接下来一直点下一步即可




    安装完成之后 win+r并输入cmd打开命令行模式

    输入node -v 查看node是否安装成功
    输入npm -v查看npm是否安装成功

    出现版本号即为安装成功 查看后不要关闭 后面步骤会继续使用命令行模式

  3. 修改安装和缓存路径

    1. 查看node安装和缓存路径

    命令行模式下 输入 npm config get prefix 和 npm config get cache查看node当前使用路径

    1. 在node安装目录新建两个文件夹 名为node_cache 、node_global

    1. 将node_modules文件夹复制一份到node_global文件夹内

    2. 使用命令修改安装路径

    注意 ：命令和文件路径之间要有一个空格隔开

    命令行模式下输入npmconfig set prefix "node_global文件的路径"

    文件路径查看方式：打开node_global文件夹、点击上方路径复制

    1. 使用命令修改缓存路径

    命令行模式下输入npmconfig set cache "node_cache文件的路径"

  4. 配置环境

    1. 右击此电脑，选择属性进入系统界面

    1. 点击高级系统设置，进入属性界面


    1. 点击环境变量进去配置界面

      1. 下方系统变量点击新建

        变量名为 NODE_PATH 变量值为 node_global文件夹中node_modules文件路径

    1. 点击上方用户变量中的path变量

    将nodejs默认配置修改为node_global的文件路径，并添加%NODE_PATH%变量

    修改前

    修改后

  5. 测试

    命令行模式下 使用命令 npm install vue -g

    注意：一定要加上-g，这是全局安装的意思，如果不加是局部安装，会安装在cmd控制台当前运行的目录下

    如图所示即为成功

  6. 报错及解决方案

    如图所示的错误

    原因：因为node安装目录文件夹的权限不够所导致

    解决方法：

    1. 找到node安装目录、右击选择属性

    1. 选择安全选项点击编辑 允许权限中的完全控制

  7. 安装hexo

    测试无报错后可以开始安装hexo

    命令行模式下 使用 npm config set registry https://registry.npmjs.org/

    此步的目的为 切换为淘宝镜像 安装hexo速度会快

    命令行模式下 使用npm install hexo-cli -g安装hexo

    如图所示即为安装成功
